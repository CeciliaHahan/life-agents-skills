# /trace — 想法演变追溯

用户输入 `/trace <topic>`，追踪该想法在 Obsidian 笔记中的出现和演变。

## 执行步骤

### 1. 解析参数

topic 从 `$ARGUMENTS` 获取。如果为空，用 AskUserQuestion 询问用户想追溯什么。

如果用户在 topic 中指定了 vault 名称（如 `in:某个vault`），只搜索该 vault。

### 2. 确定搜索范围

用户需在首次使用时告知 Obsidian vault 根目录路径。

默认搜索 vault 下的所有主要目录，包括：
- 日记/周记文件夹
- Inbox 文件夹
- 归档文件夹
- vault 根目录下的 `.md` 文件

### 3. 关键词扩展

根据 topic 联想相关的搜索词：
- 中英文翻译（如 topic 是中文则加英文，反之亦然）
- 同义词和相关表达
- 通常扩展到 3-6 个关键词即可，不要过度扩展

### 4. 搜索

对每个关键词，使用 Grep 工具在搜索范围内的 `.md` 文件中搜索：
- 使用 `glob: "*.md"` 过滤只搜 markdown 文件
- 使用 `output_mode: "files_with_matches"` 先获取匹配文件列表
- 所有关键词的搜索并行执行以提高效率

### 5. 按时间排序

从文件名中提取日期信息：
- 常见格式：`YYMMDD`（如 `250115` = 2025-01-15）
- 也可能出现 `YYYY-MM-DD` 或 `YYYYMMDD`
- 无法识别日期的文件排在最后，标注"日期未知"
- 按时间从早到晚排列

### 6. 提取相关内容

对于匹配到的文件（如果超过 15 个，优先取最近的 15 个）：
- 用 Read 工具读取文件内容
- 提取与 topic 相关的段落和上下文（不要照搬全文）
- 多个文件可以并行读取

### 7. 输出格式

输出两部分：

**第一部分：时间线**

```
## Timeline: [topic]

### 2024-03-15 — `文件夹/文件名.md`
> 相关摘录内容...

### 2024-05-20 — `文件夹/文件名.md`
> 相关摘录内容...

（按时间顺序列出所有出现）
```

**第二部分：演变总结**

```
## 演变总结

分析这个想法的发展轨迹：
- 最早何时出现，在什么语境下
- 态度/理解如何变化
- 和哪些其他概念产生了关联
- 有没有明显的转折点
```

用中文输出总结（除非用户用英文提问）。

## 注意事项

- 搜索时忽略大小写（`-i: true`）
- 如果搜索结果为零，告诉用户没找到，并建议换个关键词试试
- 文件路径显示时用相对于 vault 根目录的路径，更简洁
