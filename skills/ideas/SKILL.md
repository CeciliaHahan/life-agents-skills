# /ideas Skill

从 Obsidian vault 的笔记碎片中发现用户可能没意识到的规律，生成结构化的 idea report。

## 用法

- `/ideas` — 全扫描模式
- `/ideas <focus>` — 聚焦某个方向（如 `/ideas content`、`/ideas fitness`）

## 执行步骤

### Step 1: 广泛扫描最近的笔记

用户需在首次使用时告知 Obsidian vault 根目录路径。

按信息密度优先级，读取以下来源：

1. **最近 4-6 周的周记/周计划** — minddump、todo、内容计划、实验，想法最密集
2. **Ideas 文件夹** — 显式的 idea 收集地
3. **最近的日记 10-15 篇** — 反思、讨论、分析
4. **Inbox 最近的笔记** — 最近 5-10 个文件
5. **Drafts 文件夹** — 在写的文章草稿
6. **Knowledge 文件夹** — 最近的文件

**扫描方法：**
- 用 Glob 找到每个目录中的 `.md` 文件，按文件名日期排序取最近的
- 用 Agent 工具并行读取多个来源，加速扫描
- 如果用户传了 focus 参数，先用 Grep 筛选与 focus 相关的文件，缩小范围

### Step 2: Pattern Discovery（核心分析）

读完笔记后，识别以下 patterns：

- **反复出现的主题**：哪些话题在多篇笔记中被提及？（出现频次高的关键词/概念）
- **未完成的好奇心**：提到了但没深入的东西（出现在 todo 但没有后续笔记）
- **人名和社交线索**：笔记中反复提到的人，以及与他们相关的话题
- **工具/产品/技术提及**：用过什么、想试什么、吐槽什么
- **情绪和态度信号**：对哪些事情表现出兴奋/困惑/不满

### Step 3: 生成 Idea Report

**输出格式（中文，除非用户英文提问）：**

```
## Ideas Report — [日期]

基于最近 [N] 篇笔记的 pattern 分析

### Tools to Build
基于笔记中反复出现的痛点和需求，建议可以做的工具/项目：
1. [idea] — 来源：[哪些笔记中体现了这个需求]
2. ...

### People to Reach Out To
基于笔记中提到的人和话题交集，建议可以联系的人：
1. [人名] — 关于 [话题]，因为 [笔记中的线索]
2. ...

### Topics to Investigate
表现出好奇但还没深入的领域：
1. [话题] — 在 [N] 篇笔记中被提及，但没有专门的深度笔记
2. ...

### Things to Write
基于已有的思考积累，可以写成内容的话题：
1. [话题] — 已经在 [笔记] 中有了 [角度]，可以扩展成 [形式]
2. ...

### Books to Read
基于最近关注的话题和好奇心，推荐可能感兴趣的书：
1. [书名] by [作者] — 因为在 [笔记] 中对 [话题] 表现出兴趣
2. ...

### Movies & Shows to Watch
基于审美偏好和最近的兴趣方向：
1. [片名] — 和 [笔记] 中关注的 [话题/风格] 相关
2. ...

### Music to Listen To
基于笔记中提到的音乐偏好、情绪状态、或文化兴趣：
1. [艺术家/专辑/歌单方向] — 因为 [笔记中的线索]
2. ...

### Meta-Pattern
最近的思维趋势总结：在关注什么、兴趣怎么在变化
```

## 关键原则

- **重质不重量**：每个类别 3-5 个高质量 idea，不要堆砌
- **每个 idea 都要有笔记来源**：grounded in actual notes，不是凭空联想
- **发现用户自己没意识到的规律**：这是核心价值，不是简单总结
- **中文输出**（除非用户用英文提问则英文输出）
