# /emerge Skill

Find clusters of related ideas in the vault that could become a project, essay, or product. Show what's emerging and what notes connect to it.

核心场景：散落的想法开始聚拢，你隐约感觉到某个东西快要成形了，但还没看清全貌。

**与其他 skill 的区别：**
- `/ideas` — 发现你该做什么（行动建议清单）
- `/drift` — 发现你在无意识地靠近什么（隐性暗流）
- `/emerge` — 发现什么东西快要成形了（想法聚合 → 项目雏形）

## 用法

- `/emerge` — 无参数，全扫描

## 执行步骤

### Step 1: 广泛扫描（数据采集）

用户需在首次使用时告知 Obsidian vault 根目录路径。

按信息密度优先级读取：

1. **Ideas 文件夹** — 显式的 idea 收集地，最可能有聚合
2. **最近 4-6 周的周记/周计划**
3. **Drafts 文件夹** — 已经在写的东西 = 已经开始聚合的信号
4. **最近的日记 10-15 篇**
5. **Inbox 最近的笔记** — 最近 5-10 个文件
6. **Knowledge 文件夹** — 最近的文件
7. **其他写作文件夹** — 最近的文件

**扫描方法：**
- 用 Glob 找到每个目录中的 `.md` 文件，按文件名日期排序取最近的
- 用 Agent 工具并行读取多个来源，加速扫描
- 尽量读全文，聚合关系需要看到具体内容

### Step 2: Cluster Discovery（核心分析）

不是找单个 idea，也不是找隐性暗线，而是找 **正在聚合的想法群**：

- **多篇笔记指向同一个未命名的项目**：比如一篇笔记谈了某个痛点，另一篇记了一个技术方案，第三篇写了目标用户画像——这三篇笔记可能在指向同一个还没被命名的产品
- **已经有雏形的草稿 + 散落的素材**：Drafts 里有半成品，而其他笔记里有可以补充进去的材料、角度、案例
- **反复出现的 "想做" 信号**：在 todo、日记、idea 笔记中多次提到想做某件事，且每次都补充了新的细节——说明这件事正在从模糊变清晰
- **已有足够素材可以组装的内容**：某个话题已经积累了足够的笔记（观点、案例、引用），只差一个结构就能变成文章或视频
- **交叉引用密度高的话题**：多篇笔记之间有 `[[wikilink]]` 或概念上的相互引用

### Step 3: 输出 — Emerge Report

输出风格：每个 cluster 是一个"正在成形的东西"，需要清晰展示它由哪些笔记组成、已经走到哪一步、还差什么。

```
## Emerge Report — [日期]

基于最近 [N] 篇笔记的聚合分析

---

### Cluster 1: [给这个正在成形的东西起一个名字]

**形态：** [项目 / 文章 / 产品 / 系列内容 / 实验 / ...]

**组成笔记：**
- [笔记1] — 贡献了什么（痛点/方案/素材/角度）
- [笔记2] — 贡献了什么
- [笔记3] — ...

**已经走到哪了：** 用一段话描述这个 cluster 目前的成熟度。是刚冒头？已经有骨架？还是只差临门一脚？

**还差什么：** 具体说明要把这个 cluster 变成成品，还需要什么（一个结构、一次调研、一个决定、一次对话...）

---

### Cluster 2: [名字]
...

---

（3-6 个 clusters，按成熟度从高到低排序）

---

### 最接近 "可以开始做" 的一件事

从所有 clusters 中选出最成熟的那个，用一段话说明为什么它已经准备好了，以及建议的第一步行动。
```

## 关键原则

- **每个 cluster 必须由至少 3 篇不同笔记支撑**：不是单个 idea 的放大，而是多个碎片的聚合
- **按成熟度排序**：最接近可执行的排最前面
- **具体说明"还差什么"**：这是 /emerge 的独特价值——不只告诉你什么在聚合，还告诉你离成形还有多远
- **不要和 /ideas 重复**：/ideas 输出的是"你可以做什么"的建议清单；/emerge 输出的是"什么东西已经在自己长出来了"
- **名字要具体**：不要叫"AI 相关项目"，要叫"给创作者的 AI 写作工作流工具"
- **3-6 个 clusters**：少而精，每个都要有实质内容
- **中文输出**（除非用户英文提问）
